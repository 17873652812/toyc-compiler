# ToyC 编译器 — 计划书

> 最终目标：手写编译器，ToyC → RISC-V 32 位汇编，通过在线评测
> 语言：C++20 | 构建：CMake | 提交：GitHub + 访问令牌

---

## 一、编译器架构

```
源码 (.tc) → [Lexer] → Token流 → [Parser] → AST树 → [Codegen] → RISC-V (.s)
```

```
src/
├── defs.h      公共定义
├── token.h     Token 类型
├── lexer.h     词法分析（源码 → Token）
├── ast.h       AST 节点
├── parser.h    语法分析（Token → AST）
├── codegen.h   代码生成（AST → 汇编）
└── main.cpp    入口（读stdin/文件 → 调三层 → 写stdout）
```

## 三、v0.2 详细拆分（整个计划最关键的一步）

v0.2 是整个开发中**改动最大**的一步，拆成 3 个子步骤：

### v0.2a：Token + Lexer 扩展

新增 8 种 Token：`+` `-` `*` `/` `%` `=` `!` `,`
改 `token.h` + `lexer.h`
测试：打印 Token 列表

### v0.2b：AST + Parser 扩展

新增 AST 节点：`VarDecl` `AssignStmt` `BinaryExpr` `UnaryExpr` `IdExpr`
Parser 新增：变量声明、赋值语句、表达式（含运算符优先级）
测试：打印 AST 树

### v0.2c：符号表 + Codegen 扩展

符号表：栈式作用域、变量→栈偏移映射
Codegen：栈分配、`lw`/`sw`、算术指令（`add` `sub` `mul` `div` `rem`）
测试：编译运行，用 Spike 验证返回值

## 四、当前进度：v0.1 ✅

```c
}
```
